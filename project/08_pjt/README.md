# 08-pjt: JavaScript 기반 영화 추천 UI 구현

본 프로젝트는 JavaScript를 활용하여 HTML DOM을 동적으로 조작하고,  
영화 데이터를 기반으로 영화 목록 출력, 장르 필터링, 영화 생성 및 삭제 기능을 구현하는 것을 목표로 한다.  
제공된 UI 스켈레톤(index.html)과 데이터(raw data)를 기반으로 주어진 요구사항(F01~F05)을 단계별로 구현하였다.

---

## 1. 프로젝트 개요

- 주제: 영화 추천 커뮤니티 UI 구축  
- 목표:
  - 영화 목록을 실시간으로 렌더링하는 JavaScript 로직 구현
  - 장르별 필터링 기능을 통한 사용자 경험 향상
  - 입력 폼을 통한 영화 추가 기능
  - 영화 삭제 기능
  - 기존 HTML의 구조를 유지하면서 JavaScript만으로 UI를 동적으로 구성

---

├─ movie/

    └─

## 3. 요구사항 구현 내역

### F01. 영화 목록 출력
- div#movie-list-section 내부에 ul/li 형태로 영화 제목 목록 출력
- JavaScript로 요소(createElement) 생성 및 appendChild 사용
- 목록 갱신 시 innerHTML 비우기만 허용

---

### F02. 장르 선택 UI 및 필터링
- div#movie-genre-filter 내부에 장르 select UI 구현
- “전체” 포함
- 장르 선택 시 해당 장르 id를 포함한 영화만 렌더링

---

### F03. 영화 생성 UI
- 영화 제목 입력 input  
- 장르 선택 select  
- submit 버튼


---

### F04. 영화 추가 기능
- 제목/장르 검증 후 movieState 배열 맨 앞에 unshift로 추가
- 장르 필터는 자동으로 ‘전체’로 초기화
- 새로고침하면 사용자 추가 영화는 사라짐


---

### F05. 영화 삭제 기능
- 영화 제목 옆에 삭제 버튼 생성
- 클릭 시 movieState에서 filter로 제거 후 재렌더링
- 새로고침하면 삭제한 영화는 원본 데이터 기준으로 복구됨
- (추가된 영화는 원본 데이터가 아니므로 새로고침 시 자동 제거)

## 4. 구현 과정에서 배운 내용

### 4.1 DOM 조작의 기본 원리
- innerHTML 대신 createElement / appendChild를 통해 구조를 직접 조립하는 방식 학습
- 이벤트 기반 렌더링과 상태(state)의 개념 이해
- DOM이 “즉시 렌더링”된다는 점을 고려해 요소 생성 시점, 이벤트 등록 시점 조절

### 4.2 상태(state) 관리
- movieState 라는 별도 배열을 두어 화면에 보여주는 데이터를 관리
- 원본 movies 배열을 변형하지 않고, 화면 state만 변경하는 구조가 유용하다는 점을 체험

### 4.3 다단계 렌더링 흐름 이해
- 장르 select 변경 → state 변경 → renderMovieList() 호출 → DOM 새로 그림  
- 입력 form submit → 검증 → state 추가 → renderMovieList() 호출  
- 작은 단위 기능을 “함수 단위”로 나누는 것이 유지보수에 중요함을 배움

---

## 5. 구현 과정에서 어려웠던 점

- 제공된 스켈레톤을 유지하면서 필요한 요소들을 JavaScript만으로 배치하는 과정이 처음이라 구조 파악이 어려웠음
- genreRawData / movieRawData를 어떻게 가져와 사용하는지 이해하는 데 시간이 필요했음
- 삭제 및 추가 후 필터 상태가 바뀌었을 때 목록을 어떻게 안정적으로 유지할지 고민함
- 새로고침 후 데이터가 원래대로 돌아오는 구조를 이해하는 과정에서 state와 raw data의 차이를 명확히 구분하는 연습이 필요했음

---

## 6. 새롭게 배운 점 / 느낀 점

- HTML을 직접 수정하지 않아도 JavaScript만으로 충분히 UI를 구성할 수 있다는 점을 실습을 통해 체감할 수 있는 기회로 느꼈습니다. 
- DOM 조작과 상태 기반 렌더링을 경험하면서 프론트엔드 프레임워크(React, Vue 등)의 기본 원리를 조금 이해할 수 있게 됐습니다. 
- 작은 기능부터 하나씩 구현하고, 그 기능의 동작을 확인해가면서 확장하는 방식이 개발 효율을 크게 높여준다는 점을 배웠습니다.
이번 프로젝트는 JavaScript로 DOM을 조작하고 필터링/추가/삭제 같은 기본 기능을 구현하는 데 집중할 수 있었고,  
요구사항을 모두 충족했으며, 구현 과정에서 “상태 관리”, “함수 분리”, “DOM 이벤트 흐름”의 중요성을 명확하게 배웠습니다.
